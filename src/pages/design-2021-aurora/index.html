Inspired by Stripe

Aurora

Mesh gradients carry visual identity.
Familiar controls keep the interface clear.

Around 2021, after years of restrained white product interfaces, designers looked for stronger visual identity without rebuilding familiar interaction patterns. Aurora UI put layered, organic gradients behind otherwise conventional controls. Stripe's product pages became a prominent example of that approach.

Unlike glassmorphism, the content does not depend on background blur. Unlike neon, the color does not need to imply emitted light. The mesh establishes atmosphere and brand; solid surfaces preserve contrast.

The practical lesson is restraint: use a limited gradient field where it earns attention, keep controls recognizable, and make motion slow, optional, and compatible with reduced-motion preferences.

Principle 01

Mesh Gradients

Layer radial-gradient() blobs at different positions and sizes. Each blob is a single saturated color fading to transparent. Give each field an explicit circular radius so wide and narrow containers do not squeeze it into a pill shape. Three to five layers are a useful starting point. If the field moves, use a long cycle and provide a static reduced-motion state.

Principle 02

Protect Readability

Place text and controls on solid, high-contrast surfaces so their legibility does not change as the gradient moves. Blur and translucency are optional effects, not requirements of Aurora UI.

Principle 03

Flat Interactions

Buttons are solid fills. Inputs are clean boxes. Hover states are subtle background shifts — not glow effects. The background already provides visual emphasis. Interactive elements should prioritize readable labels, clear focus, and predictable states.

In Practice

Gradient as CTA

The mesh gradient isn't just a page background. Use it inside banners, hero sections, and call-to-action blocks. The gradient frames the message. The button is plain white. Maximum contrast, zero clutter.

What This Is Not

  • ✗ Glassmorphism — no backdrop-filter, no frosted panels
  • ✗ Neon — nothing glows, nothing emits light
  • ✗ Gradient on buttons or text — the gradient stays behind
  • ✗ Subtle tinted backgrounds — the mesh is bold and vivid
  • ✓ Mesh gradients in a defined brand or emphasis role
  • ✓ Stable content surfaces with tested contrast
  • ✓ Familiar controls and explicit interaction states
  • ✓ Optional slow motion with a reduced-motion fallback
