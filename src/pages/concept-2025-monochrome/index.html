Mono

Content is the only color.
Everything else whispers.

Monochrome records a system used in BLEX: interface chrome stays black, white, and gray, while content — images, code, and artwork — is allowed to carry color.

The chrome disappears, leaving only what matters. Instead of using accent colors to create hierarchy, Monochrome uses weight, spacing, and shades of gray to keep controls quiet and content vivid. It is a personal system, not a claim that monochrome design itself is new.

Rule 01

Radius Maps Component Roles

Radius is part of the component grammar, not a decoration chosen per screen. Large surfaces, contextual panels, direct controls, and compact statuses each keep a consistent curvature so their roles remain recognizable without color.

Rule 02

Color Belongs to Content

The interface shell uses a ten-step gray scale, from background whisper to foreground authority. Images, code, charts, and authored work keep their natural colors; controls create hierarchy through contrast instead of hue.

Rule 03

Motion Carries State

Without a colored highlight, controls confirm state through small changes in position, fill, and shadow. Direct feedback stays quick; layer changes take slightly longer. Motion explains what changed and then gets out of the way.

/* Motion confirms state without adding color */
--motion-control: 140ms;
--motion-layer: 240ms;
--ease-out: cubic-bezier(0.2, 0, 0, 1);

button:hover { background: var(--gray-100); }
button:active { transform: scale(0.98); }
Rule 04

Blur Is Reserved for Floating Tools

Content surfaces are solid and stable. A toolbar or temporary overlay may use translucent white with backdrop blur to show that it floats above content. Blur communicates layer ownership; it is not a default treatment for every card.

Limits

When the Shell Can Stay Quiet

  • ✓ Suits publishing, portfolios, readers, and tools where authored content is primary
  • ✗ Avoid when status, categories, or live data depend heavily on color
  • ✗ Gray without contrast discipline — every tone needs a readable role
  • ✗ Removing color from the content itself — only the interface shell is constrained
  • ✓ Content provides all the color
  • ✓ Every gray has a semantic role
  • ✓ Invisible chrome, visible content
  • ✓ Radius stays consistent for each component role
  • ✓ Solid content surfaces, blur only for floating tools
  • ✓ Short motion confirms state and respects reduced-motion
