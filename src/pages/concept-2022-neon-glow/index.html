Neon

Not colored. Not highlighted.
Lit from within.

Neon Glow records a violet interface direction I kept returning to: UI elements behave like light sources, not painted surfaces. It borrows the atmosphere of neon signage, then narrows it into a usable rule for digital hierarchy.

One color family. One darkness. The glow carries hierarchy inside that constraint — dim for rest, clear for action, bright for urgency. This is a personal study, not a claim that neon itself began here.

Rule 01

Glow Intensity Is Hierarchy

The same purple at three brightness levels tells three different stories. Dim is background, lit is interactive, bright is urgent. No second color needed.

Rule 02

Darkness Is the Canvas

Glow on a white background is just colored text. Glow on darkness is a light source. Same element, completely different physics. The darkness isn't absence — it's what makes the glow real.

Rule 03

One Hue, Clear States

Success, warning, and availability do not introduce new hues. The same violet is paired with explicit language, while intensity only distinguishes quiet, available, and current states.

Limits

Use the Darkness Deliberately

  • ✓ Suits media, monitoring, live events, and short focused sessions
  • ✗ Avoid for long-form reading, bright rooms, and dense productivity tools
  • ✗ text-shadow: 0 0 99px on everything
  • ✗ Rainbow RGB on every surface — one color family only
  • ✗ Glow without darkness to justify it
  • ✗ Decorative grime or particles that do not communicate hierarchy
  • ✓ One violet hue paired with explicit state labels
  • ✓ Brightness reserved for the current action
  • ✓ Dark canvas with readable, quiet body copy
